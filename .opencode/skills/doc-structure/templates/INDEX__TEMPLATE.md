## Agent Progressive Loading Guide

### Loading Strategy

| Scenario | Load Target | Est. Size |
|----------|-------------|-----------|
| First entry to project | Global index (Level 0) only | <50 lines |
| Need category details | Category README (Level 1) | ~100-200 lines |
| Need specific document | Read target file directly | 50-300 lines per doc |
| Full history review | Global index + category READMEs | <500 lines |

### Do NOT Load (unless explicitly requested)

| Content | Reason |
|---------|--------|
| Empty directories | No information value |
| `_TEMPLATE.md` files | Templates, not actual content |
| Build artifact directories | Not documentation |

### Regression Checks

Before creating a new document, the agent MUST:

1. Check global index → Read if exists; create from `templates/INDEX__TEMPLATE.md` if not
2. Check category README → Read if exists; check numbering; create from `templates/README__TEMPLATE.md` if not
3. Check for duplicates → `grep -rli "<topic>" doc/<category>/`; update existing if overlap found
4. Check for related cross-references → `grep -rl "<topic>" doc/adr/ doc/c4/ doc/research/`
