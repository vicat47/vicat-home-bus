# Beancount 集成方案 — Specification

- **Version**: 0.1.0
- **Date**: 2026-07-23
- **Author**: vicat47
- **Status**: Draft

## Overview

定义 HomeBus 与 Beancount 账本的集成方式，包括写入机制、文件隔离策略、幂等保证、校验流程和 git 同步。Beancount 作为"会计即代码"系统，无官方的写入 API——写入是"生成 `.bean` 文件分录"的行为。

## 核心决策

| 决策 | 内容 |
|------|------|
| 写入方式 | Task Executor 通过 import `beancount_writer.py` 共享库写入，非 CLI 子进程 |
| 文件隔离 | HomeBus 写入 `{ledger}/{YYYY}/0-default/homebus-{MM}.bean`，利用现有 `include "0-default/*"` glob 自动纳入，不触碰用户手写文件 |
| 库结构兼容 | 支持现有按年/月分文件夹的 Beancount 仓库结构 |
| 幂等 | 写入前扫描目标文件查已有 `homebus_event:` meta 字段 |
| 校验 | 写入后自动运行 `bean-check` |
| Git | 写入成功后 `git add` + `git commit`（flag 控制） |
| Saga 回滚 | 从 `homebus-MM.bean` 删除对应 entry 的行 + 新 git commit（非 git revert） |

## Requirements

### Functional Requirements

- **FR-1**: HomeBus 提供 `beancount_writer.py` 共享库，供 Task Executor 直接 import 完成写入（不必通过 CLI 子进程）
- **FR-2**: CLI 支持 `--ledger-path` 指定 Beancount 仓库根路径
- **FR-3**: 文件隔离——HomeBus 写入 `{ledger}/{YYYY}/0-default/homebus-{MM}.bean`，利用 `YYYY.bean` 中已有的 `include "0-default/*"` glob 自动纳入，不触碰用户手写的 `{YYYY}/0-default/{MM}.bean` 等其他文件
- **FR-4**: HomeBus 生成的 entry 携带 `#homebus` tag 标记来源
- **FR-5**: HomeBus 生成的 entry 携带 `homebus_event:` meta 字段（Event ID）用于幂等去重；携带 `homebus_time:` 记录生成时间
- **FR-6**: 写入前扫描目标文件查已有 `homebus_event:` 值，命中则跳过不重复写入
- **FR-7**: 写入后自动调用 `bean-check` 校验账本完整性
- **FR-8**: `bean-check` 失败则回滚本次写入（删除刚追加的 entry）
- **FR-9**: 写入成功后执行 `git add {homebus-file}` + `git commit`（默认行为，可通过 `--no-auto-commit` 禁用）
- **FR-10**: Saga 回滚——补偿时删除 `homebus-MM.bean` 中对应 event_id 的 entry 行，执行 `bean-check` 验证完整性，然后 `git add` + `git commit`（非 `git revert`，避免与其他 commit 产生合并冲突）
- **FR-11**: 支持 `--dry-run` 模式只生成 entry 文本不写入
- **FR-12**: MVP 查询——提供 `verify_entry` 操作：按 `event_id` 在 `homebus-MM.bean` 中查找对应 entry，返回 `{found, entry, file, line}`。供 Agent 轮询确认和集成测试断言。实现为 `beancount_writer.read_entry(event_id, ledger_path)`
- **FR-13**: v0.2 完整查询——通过 Fava REST API 支持余额查询、账目查询、科目聚合等

### Non-Functional Requirements

- **NFR-1**: 写入操作延迟应 <100ms（不含 `bean-check` 和 git 操作）
- **NFR-2**: `bean-check` 超时 10s，超时不阻塞写入结果（只告警）
- **NFR-3**: 并发写入同一 `homebus-{MM}.bean` 文件需文件锁保护

## Design

### 架构

```
Agent ──→ homebus publish ...
               │
               ▼
          HomeBus API
               │
               ├─ Grocy Adapter (HTTP)
               ├─ Homebox Adapter (HTTP)
               └─ Beancount Adapter (生成分录文本)
                      │
                      ▼
           beancount_writer.py (import)
                      │
                      ├─ 幂等检查（扫描已有 homebus_event:）
                      ├─ 追加 entry 到 homebus-MM.bean
                      ├─ bean-check ✅ / ❌ → 回滚
                      └─ git add + commit
```

### 现有账本结构兼容

用户现有 Beancount 仓库使用按年+按领域组织的目录结构：

```
~/ledger/                             ← git 管理
├── main.bean                         ← include accounts/* + YYYY/YYYY.bean
├── accounts/
│   ├── assets.bean                   ← 资产账户定义
│   ├── liabilities.bean              ← 负债账户定义
│   ├── expenses.bean                 ← 支出账户定义
│   ├── income.bean                   ← 收入账户定义
│   └── equity.bean                   ← 权益账户定义
├── 2025/
│   ├── 2025.bean                     ← include "0-default/*" 等
│   ├── 0-default/
│   │   ├── 07.bean                   ← 2025年7月日常交易（用户手写）
│   │   ├── 08.bean
│   │   ├── 09.bean
│   │   ├── 10.bean
│   │   ├── 11.bean
│   │   └── homebus-07.bean           ← HomeBus 写入（新增，glob 自动 include）
│   ├── 2-trip/                       ← 差旅
│   ├── 3-cycle/                      ← 周期费用
│   └── 5-investments/                ← 投资
└── 2024/
    ├── 2024.bean                     ← include "0-default/00.bean" 等
    ├── 0-default/
    │   └── 00.bean                   ← 2024年全年（未按月拆分）
    └── ...
```

关键设计成果：

1. **零侵入** — `2025/2025.bean` 使用 `include "0-default/*"` glob，HomeBus 写入的文件放同目录即被自动覆盖，不需要修改任何现有文件
2. **旧年份兼容** — 2024 及之前使用 `00.bean` 单文件模式，HomeBus 写入按月拆分文件不影响旧年
3. **账户体系对等** — 你的 `Expenses:Meal:Lunch`、`Liabilities:Card:CMB`、`Assets:Cash:WeChatPay` 等直接映射到 routing registry
4. **已有 `item:` 元数据** — 你已在用 Beancount meta fields，HomeBus 延续此范式并扩展 `#homebus` tag + `homebus_event:` 元数据
5. **已有 `#costflow` tag 先例** — bot.bean 使用 tag 标记来源，HomeBus 使用 `#homebus` tag 对齐同一范式

### Entry 格式

```beancount
; homebus-07.bean — Auto-generated by HomeBus, DO NOT EDIT MANUALLY
; 位于 {ledger}/2025/0-default/ 目录，由 2025.bean 的 include "0-default/*" 自动纳入

2026-07-23 * "京东" "卫生纸 x2 + 机械键盘 x1" #homebus
  homebus_event: "evt_sess1_001"
  homebus_time: "2026-07-23T10:05:03"
  Expenses:Life:DailySupplies  30.00 CNY
    item: "卫生纸"
  Assets:Inventory:Electronics  500.00 CNY
    item: "机械键盘"
  Liabilities:CreditCard:JD  -530.00 CNY

2026-07-23 * "美团" "蒙牛纯牛奶 x3" #homebus
  homebus_event: "evt_sess1_002"
  homebus_time: "2026-07-23T14:30:00"
  Expenses:Food:Groceries  60.00 CNY
    item: "蒙牛纯牛奶"
  Liabilities:CreditCard:MB  -60.00 CNY
```

### 元数据字段

| 字段 | 方式 | 说明 |
|------|------|------|
| `#homebus` | tag | 标记来源，对齐现有 `#costflow` 范式。`bean-query` 可过滤: `SELECT * WHERE tag('homebus')` |
| `homebus_event:` | meta | 事件 ID（`evt_<session>_<seq>`），幂等标识。写入前扫描同名值去重 |
| `homebus_time:` | meta | HomeBus 生成时间（ISO8601，精度到秒），区分同日多笔交易的先后顺序 |
| `item:` | meta | 延续现有记账习惯，分量级标注。从 Event 的 `items[].name` 提取 |

> **设计参考**：现有 `bot.bean` 已使用 `#costflow` tag + `tgbot_uuid:` meta + `tgbot_time:` meta 的元数据范式。HomeBus 采用相同的 tag + meta 结构，`homebus_` 前缀与 `tgbot_` 前缀对应，便于识别来源。

### 当日多笔交易

Beancount 时间精度到天。同一天的多笔交易各自对应独立的 entry，按事件发生时间在文件中物理顺序排列。无需合并。

### 幂等逻辑

```
写入前：
  for each entry in entries_to_write:
    grep "homebus_event: \"{event_id}\"" {homebus-file}
    if found → skip, return {status: "duplicate", original_entry: ...}

写入时：
  append entries to {homebus-file}

写入后：
  bean-check {ledger}
  if failed → remove appended entries, return error
  if ok → git add + commit, return success
```

### MVP 查询能力

MVP 通过 `POST /v1/query` 提供最小 Beancount 查询——按 event_id 验证写入：

```json
POST /v1/query
{
  "target": "beancount",
  "operation": "verify_entry",
  "params": {"event_id": "evt_sess1_001"}
}

// 找到
{
  "found": true,
  "event_id": "evt_sess1_001",
  "entry": "2026-07-23 * \"京东\" \"蒙牛纯牛奶 x3\" #homebus\n  homebus_event: \"evt_sess1_001\"\n  homebus_time: \"2026-07-23T10:00:00\"\n  Expenses:Food:Groceries  60.00 CNY\n    item: \"蒙牛纯牛奶\"\n  Liabilities:CreditCard:JD  -60.00 CNY",
  "file": "~/ledger/2025/0-default/homebus-07.bean",
  "line": 5
}

// 未找到
{
  "found": false,
  "event_id": "evt_sess1_001"
}
```

实现方式：`beancount_writer.py` 提供 `read_entry(event_id, ledger_path)` → 扫描当年月的所有 `homebus-MM.bean` 文件，按 `homebus_event:` meta 字段匹配。不做 Fava 集成、不做 `bean-query`。

v0.2 的完整查询（余额查询、账目查询、科目聚合）走 Fava REST API，届时 `operation` 扩展为 `balance`、`account_report` 等。

| 操作 | 版本 | 查询方式 |
|------|------|---------|
| `verify_entry` | v0.1 | 文件扫描 `homebus_event:` meta |
| `balance` | v0.2 | Fava REST API |
| `account_report` | v0.2 | Fava REST API |

### 错误处理

| 场景 | 行为 | HTTP 状态 |
|------|------|----------|
| 幂等命中（event_id 已存在）| 返回已有 entry 信息 | 200（非错误） |
| `bean-check` 失败 | 回滚写入，返回错误详情 | 500 |
| git commit 失败 | 记录 warn 日志，不阻塞写入成功状态 | — |
| 并发冲突 | 文件锁等待，超时返回错误 | 409 |
| 仓库路径无效 | 返回错误 | 400 |

## Implementation Details

### 模块架构

Beancount 写入链路不通过 CLI — Task Executor 直接 import 共享库：

```
homebus/adapters/
  beancount.py            ← 生成 .bean 分录文本（Event → Beancount 语法）
  beancount_writer.py     ← 文件 I/O + 幂等扫描 + bean-check + git（共享库）

homebus/
  executor.py             ← Task Executor，import beancount_writer 执行写入

cli/
  homebus.py              ← publish / query / status / health（Agent 用）
                           （MVP 不提供 homebus beancount write 子命令）
```

| 组件 | 文件 | 职责 |
|------|------|------|
| Beancount Adapter | `homebus/adapters/beancount.py` | entry 文本生成、分录格式映射 |
| Beancount Writer | `homebus/adapters/beancount_writer.py` | 文件 I/O、幂等扫描、bean-check 调度、git 操作、entry 查询 |
| Task Executor | `homebus/executor.py` | import `beancount_writer`，执行 Beancount 写入子任务 |
| CLI（未来扩展）| `cli/beancount.py` | 人类调试用 `homebus beancount write` 命令（thin wrapper，v0.2+）

MVP 中 `cli/beancount.py` 不存在。API Server 不通过 subprocess 调用 CLI——Exec 直接 import `beancount_writer`。未来如果需要独立 CLI 命令，`cli/beancount.py` 是 `beancount_writer.py` 的薄封装，不引入额外逻辑。

### 路由注册表集成

`routing-registry.md` 中的品类路由和渠道路由为 Beancount Adapter 提供会计分录参数：

```
purchase 事件:
  items[].category → registry.get_category_route()
    → default_beancount_account (费用/资产科目)
  store → registry.get_store_route()
    → beancount_liability (负债账户)
```

### Saga 回滚

当 Grocy 成功但 Beancount 写入失败时需要补偿：

```
1. Grocy 回滚库存（add_stock → consume_stock）
2. Beancount Adapter 已生成但未成功写入的 entry 被丢弃（Saga 回滚时状态为 "pending"）
```

当 Beancount 写入成功但 Homebox 失败时（purchase durable）：

```
Grocy ✅ → Beancount ✅ → Homebox ❌

Saga 补偿:
  1. Beancount: 从 homebus-MM.bean 删除对应 event_id 的 entry 行 + bean-check + git commit
  2. Grocy: 回滚库存
```

## Asyncio 集成

Beancount 写入链路包含同步阻塞操作（文件读写、`bean-check` 子进程、`git` 子进程）。为避免阻塞 FastAPI 事件循环：

| 操作 | 实现方式 | 说明 |
|------|---------|------|
| 文件读写 | `asyncio.to_thread(open/read/write)` | 同步 I/O 在线程池执行 |
| `bean-check` | `asyncio.create_subprocess_exec` + `asyncio.wait_for(timeout=10)` | 异步子进程，超时不阻塞 |
| `git add/commit` | `asyncio.create_subprocess_exec` | 异步子进程，与 `bean-check` 串行（先 check 后 commit） |
| 文件锁 | `asyncio.Lock`（应用层） | 保护同一 `homebus-MM.bean` 文件的并发写入，非 fcntl OS 锁 |

```python
# beancount_writer.py 核心调用模式

import asyncio

_writer_locks: dict[str, asyncio.Lock] = {}  # key: file_path

async def write_entries(entries, ledger_path, date):
    file_path = _build_file_path(ledger_path, date)
    lock = _get_lock(file_path)

    async with lock:
        # 1. 幂等扫描（文件读，to_thread）
        existing = await asyncio.to_thread(_scan_existing_events, file_path, entries)

        # 2. 追加写入（文件写，to_thread）
        await asyncio.to_thread(_append_entries, file_path, entries)

        # 3. bean-check（异步子进程）
        proc = await asyncio.create_subprocess_exec(
            "bean-check", str(ledger_path),
            stdout=asyncio.subprocess.PIPE,
            stderr=asyncio.subprocess.PIPE,
        )
        try:
            stdout, stderr = await asyncio.wait_for(proc.communicate(), timeout=10)
        except asyncio.TimeoutError:
            proc.kill()
            await _rollback_entries(file_path, entries)
            raise BeancountCheckTimeoutError()

        if proc.returncode != 0:
            await _rollback_entries(file_path, entries)
            raise BeancountCheckError(stderr.decode())

        # 4. git commit（异步子进程）
        await _git_commit(ledger_path, file_path, entries)
```

### 依赖项

不需要额外依赖——`asyncio.to_thread`（Python 3.9+）、`asyncio.create_subprocess_exec`、`asyncio.Lock` 均为标准库。

## Idempotency Path

两层幂等，events 表为权威：

```
POST /v1/events (event_id=evt_001)
         │
         ▼
    Validator: SELECT FROM events WHERE event_id='evt_001'
         │
    ┌────┴────┐
    ▼         ▼
 已存在     新事件
    │         │
    ▼         ▼
 200 +      INSERT → 继续执行
已有状态        │
                ▼
        beancount_writer: 写入前扫描 homebus-MM.bean
                │
          检查 homebus_event: "evt_001" 是否已存在
                │
           ┌────┴────┐
           ▼         ▼
        已存在    未找到
           │         │
           ▼         ▼
       告警日志   正常追加
       不重复写
```

| 层级 | 权威性 | 行为 |
|------|--------|------|
| events 表 | **权威源** | 命中去重 → 直接返回。INSERT 冲突触发 `sqlite3.IntegrityError` |
| .bean 文件 | **防御层** | 发现不一致 → 记录 WARN 日志 + 不静默跳过，等待人工排查 |
